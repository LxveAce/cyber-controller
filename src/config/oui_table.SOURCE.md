# `oui_table.tsv.gz` — provenance

Compact `PREFIX<TAB>Organization` mapping (gzipped, sorted) of the IEEE MAC-address registry, used by
`src/core/oui.py` to label scanned devices with their manufacturer. Covers all three IEEE block sizes with
**variable-length prefixes**: 6 hex = 24-bit **MA-L**, 7 hex = 28-bit **MA-M**, 9 hex = 36-bit **MA-S**. The
runtime does a **longest-prefix match** (36→28→24) so a device in a sub-assigned block resolves to its real
vendor, not the /24 block administrator.

- **Source:** the public IEEE Registration Authority data, via the Wireshark `manuf` file —
  <https://www.wireshark.org/download/automated/data/manuf> (generated by `tshark -G manuf` from the IEEE
  registry; the IEEE site itself blocks automated `.csv` access). Third column (full vendor name) is used.
- **Retrieved:** 2026-07-23. **Entries:** 57,700 (39,549 MA-L / 6,487 MA-M / 11,664 MA-S).
- **Generated by:** `python scripts/gen_oui_table.py --manuf path/to/manuf` (`prefix → organization` only —
  no short names, no addresses).
- **Contents:** the prefix→organization *assignments* only. This is factual registry data (as bundled by
  Wireshark, nmap, aircrack-ng, …); redistributed with attribution to IEEE.
- **Refresh:** re-download `manuf` and re-run the generator, or load a fresh copy at runtime via
  `oui.load_manuf(path)` (also variable-length) / `oui.load_ieee_csv(path)` (MA-L csv) — no rebuild needed.
